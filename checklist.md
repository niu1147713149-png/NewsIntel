# 验收清单 (Checklist)

> **用途**: 确保每个开发任务满足质量标准
> **基于**: .trae/rules/quality-gate.md

## 1. 代码质量 (Code Quality)

- [ ] **代码风格**:
    - [ ] Python 代码符合 PEP 8 (使用 Ruff 检查)
    - [ ] TypeScript 代码无 lint 错误 (使用 ESLint)
    - [ ] 变量命名规范 (英文, 清晰)
- [ ] **类型安全**:
    - [ ] Python 函数有完整 Type Hints
    - [ ] TypeScript 启用 Strict Mode, 无 `any` 类型
- [ ] **注释与文档**:
    - [ ] 复杂逻辑有注释说明
    - [ ] API 接口有 OpenAPI/Swagger 文档
    - [ ] 新增功能更新了 README 或相关文档

## 2. 功能测试 (Functional Testing)

- [ ] **单元测试**:
    - [ ] 核心业务逻辑覆盖率 > 80%
    - [ ] 包含边界条件测试 (空值, 异常值)
- [ ] **集成测试**:
    - [ ] API 接口返回状态码正确 (200, 400, 401, 404, 500)
    - [ ] 数据库读写正确
    - [ ] 外部 API 调用有 Mock
- [ ] **性能测试**:
    - [ ] API 响应时间 P95 < 500ms
    - [ ] 数据库查询无 N+1 问题
    - [ ] 关键路径有索引支持

## 3. 安全性 (Security)

- [ ] **数据安全**:
    - [ ] 敏感配置 (API Key, DB Password) 使用环境变量
    - [ ] 用户密码加密存储 (bcrypt)
- [ ] **输入验证**:
    - [ ] 所有 API 输入参数经过 Pydantic 验证
    - [ ] SQL 注入防护 (使用 ORM)
    - [ ] XSS 防护 (React 自动处理 + 清洗)
- [ ] **权限控制**:
    - [ ] 接口有正确的认证鉴权 (JWT)
    - [ ] 垂直/水平越权检查

## 4. AI/NLP 模块 (AI Specific)

- [ ] **模型加载**:
    - [ ] 模型文件缓存正确
    - [ ] 显存/内存占用在预算内
- [ ] **推理结果**:
    - [ ] 情感分析输出包含 Label 和 Score
    - [ ] 分类结果置信度 > 0.3
    - [ ] 异常文本 (空, 乱码) 处理不崩溃
- [ ] **兜底机制**:
    - [ ] 模型服务不可用时有降级处理

## 5. 前端体验 (UI/UX)

- [ ] **响应式**:
    - [ ] 移动端/桌面端布局正常
- [ ] **交互**:
    - [ ] 加载状态 (Loading/Skeleton)
    - [ ] 错误提示友好
- [ ] **视觉**:
    - [ ] 符合设计规范 (颜色, 字体)
    - [ ] 暗色模式显示正常
