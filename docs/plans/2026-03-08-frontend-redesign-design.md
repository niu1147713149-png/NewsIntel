# 2026-03-08 前端重设计方案

## 背景

当前前端已经具备完整的功能闭环：行情同步、登录认证、个人中心、自选股、告警、通知中心。但视觉表达存在几个明显问题：

- 页面信息像“功能卡片堆叠”，层级不够稳定
- 首页、通知、个人中心之间缺少统一设计语言
- 部分文案与交互呈现出明显的 AI 生成痕迹
- 缺少一套稳定的设计系统来约束后续演进

本次重设计目标不是简单“换皮”，而是重建一套适合 NewsIntel 的产品界面语言。

---

## 设计目标

1. 降低 AI 痕迹，建立更成熟、更像真实产品团队产出的界面风格
2. 统一首页、通知中心、个人中心、自选股、告警页的视觉系统
3. 强化“新闻情报 + 市场观察 + 操作工作台”三种能力的协同
4. 为后续新增页面提供可复用的布局、卡片、标签、筛选、空态、状态规范

---

## 路线判断

根据需求，适合采用“创意 + 规范结合”路线：

- 使用 `ui-ux-pro-max` 负责整体设计系统、版式、配色、层级、交互规范
- 后续进入实现阶段时，再让实现型技能基于这套设计系统落代码

这意味着当前阶段先出完整方案，不直接开始重构代码。

---

## 三套方向方案

### 方案 A：Signal Desk

**定位**

现代产品风与媒体情报风融合的“情报工作台”。

**参考气质**

- Linear 的秩序感
- Vercel 的现代深色质感
- The Economist 的内容判断气质

**核心视觉特征**

- 深石墨蓝背景，不使用纯黑
- 卡片更少，但结构更明确
- 大标题克制、正文密度适中
- 强调模块边界和阅读节奏，而不是堆功能
- 高亮色只服务于状态和行动，不做大面积装饰

**适合场景**

- 首页总览
- 通知中心
- 自选股/告警页
- 需要边看边操作的页面

**优点**

- 产品感最强
- 易于长期迭代
- 最适合 NewsIntel 当前“内容 + 操作”混合型产品

**风险**

- 需要从布局系统到组件系统一起改，不能只修局部

---

### 方案 B：Editorial Intelligence

**定位**

更偏媒体情报风的分析平台，强调“读懂事件，再理解市场”。

**参考气质**

- The Economist
- Notion AI
- Arc / Readwise 风格的信息阅读体验

**核心视觉特征**

- 更强调排版和内容块，而不是仪表盘感
- 头条、摘要、事件线、观点标签占主导
- 图表和数据更多作为辅助证据出现
- 页面更像“分析简报”而不是“管理后台”

**适合场景**

- 首页内容流
- 新闻详情
- 事件分析页

**优点**

- 高级感强
- AI 痕迹最少
- 非常适合体现“情报”属性

**风险**

- 功能型页面（告警、筛选、管理）容易被弱化
- 如果未来强调操作效率，延展性稍弱

---

### 方案 C：Research OS

**定位**

更偏现代 SaaS 的研究面板，把媒体气质压低，强调效率和稳定性。

**参考气质**

- Stripe Dashboard
- Vercel Dashboard
- 专业数据工具类产品

**核心视觉特征**

- 栅格更规整
- 指标卡、表格、筛选条、状态标签非常强
- 阅读感较弱，工具感更强

**优点**

- 最稳妥
- 最容易快速重构
- 组件复用率最高

**风险**

- 可能还是会接近“普通后台”
- 不能充分体现项目的新闻/情报差异化

---

## 推荐方案

### 推荐：方案 A（Signal Desk）

这是最适合当前项目阶段的主方案。

理由：

1. 项目不是纯新闻站，也不是纯量化终端，而是“情报工作台”
2. 需要同时承载内容阅读、市场判断和操作入口
3. 可以吸收方案 B 的内容表达方式，但不牺牲操作效率
4. 更容易形成一套完整的设计系统，并覆盖后续所有页面

## 新版视觉系统建议

### 1. 视觉关键词

- Calm
- Precise
- Editorial
- Analytical
- Premium

### 2. 配色建议

**主背景**

- `#0B1020`
- `#111827`
- `#0F172A`

**卡片背景**

- `#111827`
- `#172033`
- `#1B2437`

**边框**

- `#243041`
- `#2A3648`

**主高亮**

- 蓝：`#4F8CFF`
- 青：`#2DD4BF`

**语义色**

- 利好 / 成功：`#22C55E`
- 利空 / 风险：`#EF4444`
- 中性 / 等待：`#F59E0B`
- 信息 / 状态：`#60A5FA`

### 3. 字体建议

**方案建议**

- 标题：`Inter` / `Plus Jakarta Sans`
- 正文：`Inter`
- 数字：沿用等宽数字样式

**规则**

- 标题更短、更克制
- 正文行高 1.6 左右
- 数据区避免大段说明文字

### 4. 圆角与阴影

- 全局主圆角：`16px`
- 小标签/按钮：`10px`
- 阴影极弱，主要靠边框和层次对比，不靠发光效果

### 5. 动效规则

- 所有 hover / filter / collapse 动效控制在 `180ms - 220ms`
- 不使用夸张位移动效
- 通知分组折叠使用 opacity + translateY 轻微过渡

## 关键页面改版思路

### 1. 首页

**目标**

从“功能卡片集合”改为“今日情报工作台”。

**新版结构**

1. 顶部 Hero
   - 一句话价值主张
   - 今日状态摘要
   - 快速行动按钮

2. 重点情报区
   - 今日重点事件
   - 市场情绪摘要
   - 最新触发告警 / 通知

3. 工作区
   - 自选股观察列
   - 告警状态列
   - 行情同步与系统状态列

4. 新闻流区
   - 重要新闻卡片
   - 可按主题切换

**要避免**

- 过多等权重卡片并排
- 每个模块都像后台组件

---

### 2. 通知中心

**目标**

从“告警列表页”升级为真正的消息收件箱。

**新版结构**

- 顶部摘要条：当前结果、未读比例、最新触发
- 左侧 / 上方筛选条：scope、sort、ticker
- 主体：按 ticker 分组的 inbox 结构
- 组头：最新摘要、未读数、快捷操作
- 组内：可折叠通知流

**设计原则**

- 先判断，再展开，再操作

---

### 3. 个人中心

**目标**

弱化“后台个人资料页”，强化“账户与观察偏好页”。

**新版结构**

- 账户卡
- 行为概览卡（自选、告警、通知）
- 最近活动
- 偏好与安全入口

---

### 4. 自选股 / 告警页

**目标**

从普通列表升级为 watchdesk。

**新版结构**

- 顶部：筛选与批量操作
- 主体：卡片与表格式混合布局
- 侧边 / 上方：状态与风险摘要

## 改造顺序建议

### Phase 1：设计系统与首页

- 先定义配色、字体、圆角、阴影、按钮、标签、卡片
- 然后重做 `frontend/src/app/page.tsx`

### Phase 2：通知中心

- 基于新设计系统重做 `frontend/src/app/notifications/page.tsx`
- 通知组件完全纳入统一卡片与分组体系

### Phase 3：个人中心 / 自选股 / 告警

- 让账户页、watchlist、alerts 共用同一套组件语言

### Phase 4：全局清理

- 清理其余页面乱码文案
- 统一空态、错误态、加载态

---

## 明天接手的最优起点

明天最建议直接从这 3 步开始：

1. 先确认采用 **方案 A：Signal Desk**
2. 先抽设计系统基础组件，不急着铺所有页面
3. 先重做首页，再推进通知中心

## 明确结论

- 主方案：**A / Signal Desk**
- 辅助借鉴：吸收 **B / Editorial Intelligence** 的内容排版能力
- 不建议把 **C / Research OS** 作为主方向

如果进入实现阶段，应优先重构：

1. `frontend/src/app/page.tsx`
2. `frontend/src/app/notifications/page.tsx`
3. `frontend/src/app/profile/page.tsx`
4. `frontend/src/app/watchlist/page.tsx`
5. `frontend/src/app/alerts/page.tsx`
